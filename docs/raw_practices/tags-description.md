Descriptions des Tags (Méthodologies et Pratiques)

# Category Interaction & Communication Style

## Tag : Written / Async-Ready

### Description
Can be accomplished effectively through writing or offline contribution without immediate real-time presence

### Justification
The shift to written/asynchronous communication lowers the barrier of social anxiety (non-natives, juniors) and allows cognitive time to formulate a response

### Exemples de pratiques
TDD, Users stories,, A3 Report

---

## Tag : Visual / Tactile

### Description
Uses physical or digital boards, cards, or diagrams to communicate rather than just conversation

### Justification
Allows breaking the abstraction of unfinished specifications by materializing the workflow, ideal for profiles that need concreteness

### Exemples de pratiques
Kanban, Storry mapping, Example mapping

---

## Tag : Verbal-Heavy

### Description
Success that relies heavily on speaking up, debating, or verbalizing thoughts in real-time

### Justification
Recommended only for extroverted profiles or in cases of blockage requiring maximum communication bandwidth

### Exemples de pratiques
Daily Scrum, Sprint Review, Backlog refinement, Gemba Walk

---

## Tag : Remote-Friendly

### Description
Well suited for remote work

### Justification
Standardizing "Remote-Friendly" practices avoids the creation of isolated sub-teams (silos) and enforces a formality that keeps information accessible to everyone, regardless of location

### Exemples de pratiques
Code review

---

## Tag : Co-located / On-Site

### Description
Physical presence required to defuse delays and facilitate communication

### Justification
Sometimes, friction arises from an overuse of remote work. Suggesting an "On-site" practice (e.g., meeting one day a week) helps to unlock systemic latencies that are impossible to resolve remotely

### Exemples de pratiques
Impact Mapping, Retrospective

---

# Category Social Density & Energy

## Tag : Small Group / Pair

### Description
Done in intimate groups of 2-3 people.

### Justification
Working in pairs (e.g., Pair Programming) provides psychological safety for juniors, ensures real skill transfer, and bypasses silos

### Exemples de pratiques
Pair Programming, , Three Amigos, 5 Whys

---

## Tag : Whole Crowd

### Description
Requires the presence and attention of the entire team (and sometimes stakeholders)

### Justification
Necessary for global alignment, but identified as highly "energy-consuming". The tool must limit exposure to this tag for introverted profiles

### Exemples de pratiques
Sprint Planning, , Lean Coffee, Retrospective

---

## Tag : Solo-Capable

### Description
Can be performed individually, even if the result is shared later.

### Justification
Restores autonomy and technical flow to experts hindered by slow group synchronization and frequent interruptions

### Exemples de pratiques
Refactoring,, Continuous Integration

---

# Category Structure & Cognitive Load

## Tag : Structured / Facilit.

### Description
Has a clear agenda, a facilitator, and specific steps (not a free-for-all).

### Justification
A strong framework (clear facilitator) gives meaning to the ritual and avoids bureaucratic drift ("mechanical scrum")

### Exemples de pratiques
RAID Analysis,WSJF,MoSCoW

---

## Tag : Time-Boxed

### Description
Has a strict, short duration (often <15 mins or rigid intervals).

### Justification
Limits time loss and maintains cognitive engagement, particularly in the face of technical or institutional latencies

### Exemples de pratiques
Daily Scrum,Example Mapping

---

## Tag : Gamified

### Description
Uses distinct rules, turns, voting mechanisms, or physical props with depersonalization of debates through gaming or abstraction

### Justification
Depersonalize the debate (clear rules, maps) for analytical profiles ("blue/red") who reject an overly emotional approach

### Exemples de pratiques
Planning Poker,T-Shirt Estimation

---

## Tag : Spontaneous / Improv

### Description
Requires thinking on your feet, answering unexpected questions on-the-fly, or brainstorming from scratch

### Justification
Meets the need for freedom of action for experts faced with the heaviness of the process

### Exemples de pratiques
Swarming,Mob Programming

---

# Category Emotional & Conflict Exposure

## Tag : High Visibility

### Description
Practice requiring exposure and presentation of one's work in front of a large or hierarchical group

### Justification
Practices (e.g., public demo of a failure) that trigger impostor syndrome

### Exemples de pratiques
Sprint Review,Sprint Retrospective,Tigers, Paper Tigers, and Elephants

---

## Tag : Consensus-Driven

### Description
The activity cannot end until everyone agrees (or compromises)

### Justification
Requires alignment that can exhaust impatient personalities, but is necessary for quality (DOD)

### Exemples de pratiques
DoD, Team Canvas, Sprint Planning

---

## Tag : Critical / Introspect.

### Description
Direct analyse and evaluation of past work or peers

### Justification
Allows post-mortem analysis, but the tool should only suggest this tag at appropriate times to avoid emotional overload

### Exemples de pratiques
Value Stream Mapping,Graveyard of Gloom, Code Review, Sprint Retrospective

---

## Tag : Role-Fluid

### Description
Rotation of administrative duties to prevent socio-technical erosion

### Justification
Highly technical profiles hate being locked into a Scrum Master role that distances them from code, leading to technical erosion

### Exemples de pratiques
Round-Robin SM

---

# Category Technical Cadence & Feedback Loop

## Tag : Fast-Feedback

### Description
Ultra-short feedback loop limmiting latency and stagnation

### Justification
Frustrated by long wait times (4-hour builds). Includes practices ensuring an ultra-short feedback loop to maintain cognitive engagement

### Exemples de pratiques
Continuous Deployment,TDD

---

## Tag : User-Feedback Orien.

### Description
Direct, rapid or frecquent contact with the end user

### Justification
Need for direct and rapid contact and feedback from the user to meet a need for recognition and adaptation

### Exemples de pratiques
Gamba Walk, Sprint Review (si vrais users présents), A/B Testing, Beta Testing release, User Story Mapping

---

# Category Knowledge & Technical Continuity

## Tag : Documented / Traceable

### Description
Creation of a long-term, searchable memory or the project and team

### Justification
Vital need for order and documentation to alleviate the anxiety of chaos

### Exemples de pratiques
Architecture Decision Records (ADR),Wiki Updates.

---

## Tag : Maintenance-Aware

### Description
Explicitly taking into account technical debt and legacy

### Justification
Ignoring "legacy" creates strong friction. A "Maintenance-Aware" tag helps characterize practices that officially integrate technical debt

### Exemples de pratiques
Tech Debt Backlog, Ressources Allocation

---
