# Story 1.7: Remove Pending Team Invitations

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a team member,
I want to remove an invited member who has not yet created an account,
so that I can manage pending invitations effectively and correct mistakes.

## Acceptance Criteria

1. **Given** I am on the Team Members panel
   **When** I see a member with "Pending" (invited but not accepted) status
   **Then** I see a [Remove/Cancel Invite] button next to their name

2. **Given** I click the [Cancel Invite] button for a pending member
   **When** the action completes
   **Then** the invitation is revoked and removed from the list
   **And** an event is logged: `{ action: "invite.cancelled", inviteId, teamId, timestamp }`

## Dev Notes

### Technical Requirements
- **API error format:** `{ code, message, details?, requestId }`
- **RequestId:** generated by middleware and returned in every response (success + error)
- **Team isolation:** every query must include `teamId` filter. Verify the invite actually belongs to the provided `teamId`.
- **Event logging:** Every mutation logged in `events` table (transactional). Ensure NO redundant action field inside the payload as established in Story 1.6 review. Note that timestamps should be named <name>At with Z and be ISO strings.
- **TypeScript strict:** no `any`, no implicit types.

### Architecture Compliance
- **Layered backend:** routes → controllers → services → repositories
- **Repositories are DB-only:** no business logic or branching
- **Transactions required:** Invite deletion + event log MUST be atomic.
- **REST Paths:** Plural nouns `/api/v1/teams/:teamId/invites/:inviteId`, response formats must align with previous implementations.

### Library / Framework Requirements
- **Prisma 5.0+**
- **React 18.2+**, **TailwindCSS 3.3+** for styling consistency (use existing button variants).
- **Zustand** for state management (update the team members slice correctly).
